When would you want to add a remote repository rather than keeping all your work local?
For backup reason, to share code with remote developer, to be able to retrieve code from different computer.

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?
To allow me to check changes before pulling them

Describe the differences between forks, clones, and branches. When would you use one instead of another?
- Fork is clone on GitHub repository only
- Clone can happen between any repository
- Branch happens on a single repository

What is the benefit of having a copy of the last known state of the remote stored locally?
To get the latest change of the project and to have the possibility to merge  new changes when offline

How would you collaborate without using Git or GitHub? What would be easier, and what would be harder?
I don't know how we have done before Git and GitHub before!!!

When would you want to make changes in a separate branch rather than directly in master? What benefits does each approach have?
When working with others, to avoid pushing unwanted changes, to have the opportunity to merge new changes locally on my new branch, to submit pull request on the new branch
